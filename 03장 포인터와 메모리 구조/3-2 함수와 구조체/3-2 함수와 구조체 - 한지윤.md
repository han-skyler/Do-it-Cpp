# 3-2. 함수와 구조체

<aside>

함수(Function)와 구조체(Structure)의 기본 개념과 활용법을 다룹니다. 
반복되는 코드를 효율적으로 관리하기 위한 함수의 정의 및 호출 방법을 배우고, 다양한 자료형의 데이터를 하나로 묶어 관리하는 구조체의 선언과 이를 함수의 매개변수로 활용하는 최적화 기법을 학습하는 것이 목적입니다.

# 📂 함수 만들기

함수는 특정 작업을 수행하는 코드의 집합

함수를 선언할 때는 다음 4가지 요소가 필요합니다.

- 반환 형식(Return Type)
    - 함수가 처리를 마치고 돌려줄 값의 자료형입니다. 반환값이 없으면 `void` 를 사용
- 함수 이름
    - 호출 시 사용할 식별자
- 매개변수(Parameter)
    - 함수 내부에서 사용할 데이터를 전달받는 지역 변수
- 함수 몸체
    - 함수의 실제 기능을 구현하는 부분

```cpp
int add(int _x, int _y) { // int: 반환 형식, add: 이름, (int _x, int _y): 매개변수
    int result = _x + _y; // 함수 몸체
    return result;
}
```

# 📂 함수 매개변수 사용하기

## 🔎 일반 변수 전달(Call by Value)

일반 변수를 매개변수로 전달하면 값이 복사되어 전달됩니다.
함수 내부에서 이 값을 수정해도 함수 밖의 원본 변수는 아무런 영향을 주지 않습니다.

```cpp
void func(int val) {...}

int a = 1;
func(a); // Call by Value
```

## 🔎 포인터를 매개변수로 사용 (Call by Address)

원본 데이터를 직접 수정해야 할 경우 포인터를 사용합니다.

- 호출 시
    - 변수의 주소(`&a`)를 전달
- 정의 시
    - 매개변수를 포인터 타입(`int *val`)으로 선언
- 내부 동작
    - 역참조 연산자(`*`)를 통해 메모리 주소에 직접 접근하여 값을 수정

```cpp
void func(int *val) {...}

int a = 1;
func(&a); // Call by Value
```

<aside>

### Call by Reference와 다른점?

| 구분 | Call By Address (주소에 의한 호출) | Call By Reference (참조에 의한 호출) |
| --- | --- | --- |
| **도구** | **포인터(`*`)**를 사용합니다. | **레퍼런스(`&`)**를 사용합니다. |
| **전달 내용** | 변수의 **메모리 주소값**을 복사해서 전달합니다. | 변수의 **별칭(alias)**을 전달합니다. |
| **함수 선언** | `void func(int *ptr)` | `void func(int &ref)` |
| **함수 호출** | `func(&value)` (주소 연산자 사용) | `func(value)` (일반 변수처럼 호출) |
| **내부 접근** | **역참조 연산자(`*`)**를 써야 값에 접근 가능합니다. | **일반 변수**처럼 이름만으로 즉시 접근 가능합니다. |

### 용도 및 역할

**Call by Value**

- 용도: 함수 내부에서 데이터를 사용하되, 원본 데이터를 보호해야할 때 사용합니다.
- 역할: 인자로 전달된 값을 복사하여 지역 변수로 만듭니다. 함수 내에서 값을 바꿔도 호출한 곳의 원본에는 영향을 주지 않습니다.

**참고**

- Call by Value의 경우 복사를 통해 값을 넘기므로 데이터가 큰 객체라면 비효율적입니다.
보통 Call by Reference를 사용합니다
- 원본 객체가 손실되는 것을 원치 않을 경우 const를 붙여 상수성을 확보합니다.

**Call by Address**

- 용도: 포인터를 통해 원본의 주소를 전달하여 원본을 수정하거나, NULL 값 전달이 필요할 때 사용합니다.
- 역할: 메모리 주소값을 복사하여 전달하며, 역참조 연산자를 통해 원본에 접근합니다

**Call by Reference**

- 용도: 함수 내부에서 원본 데이터를 직접 수정해야 하거나, 복사 오버헤드를 피하고 싶을 때 사용합니다.
- 역할: 원본 변수에 **별칭(Alias)**을 붙여 직접 접근합니다. **C++에서 가장 권장**되는 방식입니다.
</aside>

## 🔎 배열을 매개변수로 사용

배열을 전달할 때는 배열의 이름(시작 주소)이 전달됩니다. 따라서 함수 내에서 배열 요소를 변경하면 원본 배열도 변경됩니다.

# 📂 구조체 만들기

## 🔎 구조체의 정의와 선언

구조체는 하나 이상의 변수를 묶어 새로운 사용자 정의 자료형을 만드는 도구입니다. 서로 다른 자료형(이름, 나이, 키 등)을 하나의 그룹으로 관리할 수 있습니다.

## 🔎 멤버 접근 및 초기화

```cpp
struct Person {
    string name;
    int age;
    float height;
};
```

- 초기화
    - 중괄호`{}` 를 사용하여 한 번에 값을 대입할 수 있음
    - `Person adult = {"Brain", 24, 180.0f};`
- 멤버 접근
    - 점(`.`) 연산자를 사용
    - `adult.name = “brain”`

## 🔎 구조체와 함수

구조체도 함수의 매개변수로 쓰일 수 있습니다. 하지만 구조체 크기가 클 경우 성능 저하를 막기 위해 구조체 포인터를 사용하는 것이 권장됩니다.
