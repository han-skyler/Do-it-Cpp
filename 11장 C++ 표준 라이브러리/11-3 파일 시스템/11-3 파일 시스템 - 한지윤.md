# 11-3 파일 시스템

# 파일 시스템이란 무엇일까?

파일 시스템은 데이터를 저장하고 관리하는 체계입니다. 운영체제마다 디렉터리, 파일 종류, 권한, 상태, 속성 등의 구조가 달라, 프로그래밍 시 사용하는 헤더 파일과 방법이 다릅니다. C++ 표준 라이브러리는 이러한 차이를 해결하기 위해 파일 시스템 관련 형식과 함수를 지원합니다.

- **`fstream`과 파일 시스템 라이브러리의 차이**:
    - `fstream`: 파일 **내용**을 읽고 쓰는 역할. 파일 이름, 위치 등 **메타데이터는 수정 불가**.
    - 파일 시스템 라이브러리: 파일에 관한 **정보(메타데이터)에 접근**하지만, 파일 **자체를 읽을 수는 없음**.

# C++ 파일 시스템 라이브러리

C++17부터 표준 라이브러리에 파일 시스템 라이브러리가 도입되었습니다.

- 모든 형식과 함수는 `std::filesystem` 네임스페이스에 속합니다.
- 사용하려면 `<filesystem>` 헤더 파일을 포함해야 합니다.
- 파일과 디렉터리 조작에 유용한 기능을 제공합니다.

## 절대 경로와 상대 경로

모든 파일에는 고유한 위치를 나타내는 **경로**가 있습니다.

- **절대 경로 (absolute path)**: 최상위 디렉터리(루트)부터 원하는 파일까지의 전체 경로.
    - Windows: `C:\\`, `D:\\` 등.
    - Linux: `/`.
    - 경로의 맨 앞이 루트로 시작하면 절대 경로입니다.
- **상대 경로 (relative path)**: 현재 위치를 기준으로 한 경로.
    - 예시: 현재 실행 중인 프로그램의 절대 경로가 `/foo/bar`일 때, `a/b`는 `/foo/bar/a/b`를 의미합니다.

## 경로를 나타내는 `path` 객체

파일 시스템 라이브러리에서 파일이나 디렉터리를 다루는 모든 함수는 `path` 객체를 매개변수로 받습니다.

1. 원하는 경로에 있는 파일/디렉터리의 `path` 정의.
2. 해당 `path`로 파일/디렉터리 정보 수집.
- **주의**: `path` 객체만으로는 실제 파일 존재 여부를 알 수 없습니다. `path` 클래스는 경로를 나타낼 뿐 실제 파일을 지칭하지 않습니다. 존재 여부 확인을 위해서는 `exists` 함수를 사용해야 합니다.
- `exists` 함수 원형:
    
    ```cpp
    bool exists(const std::filesystem::path& p)
    ```
    

# 파일 시스템 활용하기

C++17의 `std::filesystem`은 boost의 `filesystem`을 표준에 병합한 것입니다.

- `std::filesystem`을 사용하려면 C++17 이상의 컴파일러가 필요합니다.

```cpp
#include <iostream>
#include <string>
#include <filesystem> // 파일 시스템 헤더 파일
#include <fstream>    // 파일 입출력 헤더 파일

using namespace std;
namespace fs = filesystem;

int main() {
    // 1. 디렉터리 생성
    fs::create_directories("MyDirectory/");

    // 2. 파일 생성과 쓰기
    ofstream outFile("MyDirectory/myFile.txt");
    outFile << "Hello, FileSystem Library!" << endl;
    outFile.close();

    // 3. 디렉터리 내의 파일 확인
    cout << "Files in MyDirectory:\\n";
    for (const fs::directory_entry& entry : fs::directory_iterator("MyDirectory")) {
        if (entry.is_regular_file()) {
            cout << entry.path().filename() << endl;
        }
    }

    // 4. 파일 읽기
    ifstream inFile("MyDirectory/myFile.txt");
    string line;
    while (getline(inFile, line)) {
        cout << line << endl;
    }
    inFile.close();

    // 5. 파일과 디렉터리 삭제
    fs::remove_all("MyDirectory");

    return 0;
}
```

**실행 결과**:

```
Files in MyDirectory:
myFile.txt
Hello, FileSystem Library!
```

### 1. 디렉터리 생성

- `fs::create_directories("MyDirectory/");`
- `create_directories` 함수는 지정된 경로에 디렉터리를 생성합니다.
- 경로에 중간 단계의 디렉터리가 없으면 함께 생성하며, 이미 존재해도 오류가 발생하지 않습니다.

### 2. 파일 생성과 쓰기

- `ofstream outFile("MyDirectory/myFile.txt");`
- `outFile << "Hello, FileSystem Library!" << endl;`
- `outFile.close();`
- `ofstream`을 사용하여 `MyDirectory` 안에 `myFile.txt` 파일을 생성하고 내용을 씁니다.
- **파일 닫기 (`outFile.close()`의 중요성)** :
    - 파일 작업 후 `close()`를 호출하여 파일을 제대로 닫는 것이 중요합니다.
    - 메모리 정리 및 파일 접근 문제 방지를 위해 필요합니다.
    - `ofstream`의 소멸자가 자동으로 파일을 닫아주지만, 명시적 호출은 코드 가독성 및 유지보수 측면에서 권장됩니다.

### 3. 디렉터리 탐색

- `for (const fs::directory_entry& entry : fs::directory_iterator("MyDirectory")) { ... }`
- `directory_iterator`를 사용하여 `MyDirectory` 내의 모든 파일과 디렉터리 정보를 순회합니다.
- `directory_entry` 객체는 각 항목에 대한 참조를 나타냅니다.
- `entry.is_regular_file()`: 현재 항목이 일반 파일인지 확인합니다.
- `entry.path().filename()`: 파일의 전체 경로(`entry.path()`)에서 파일 이름만 추출하여 출력합니다.

### 4. 파일 읽기

- `ifstream inFile("MyDirectory/myFile.txt");`
- `string line;`
- `while (getline(inFile, line)) { cout << line << endl; }`
- `inFile.close();`
- `ifstream`을 사용하여 `MyDirectory/myFile.txt` 파일을 읽기 모드로 엽니다.
- `getline(inFile, line)` 함수는 파일에서 한 줄씩 읽어 `line` 변수에 저장하고 화면에 출력합니다. 파일의 끝에 도달하면 `false`를 반환하여 반복을 종료합니다.
- 파일 작업 완료 후 `inFile.close()`를 호출하여 파일 자원을 해제합니다.

### 5. 파일 삭제

- `fs::remove_all("MyDirectory");`
- `remove_all` 함수는 지정한 디렉터리(`MyDirectory`)와 그 안에 포함된 모든 하위 파일 및 디렉터리를 재귀적으로 완전히 삭제합니다.
- 모든 요소를 삭제하므로 사용에 주의해야 합니다.
