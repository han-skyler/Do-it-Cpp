</aside>

C++ 언어의 역사는 AT&T 벨 연구소의 비야네 스트롭스트룹이 1979년에 착수한 ‘C with Classes’ 연구에서 출발했다.

**C++ 언어 규약 초안**

<aside>

- 클래스
- 상속 클래스
- 멤버 함수와 변수 접근 제어(public, private)
- 생성자와 소멸자
- 프렌드(friend) 클래스
- 형식 검사와 함수 인자 변환
- 인라인 함수
- 기본 인잣값, 연산자 오버로딩

## C++ 언어의 탄생

스트롭스트룹은 C++ 언어 규약을 더 정교하게 다듬었다. 그러면서 C++는 C언어의 확장이 아닌 하나의 언어로서 제 모습을 갖추었다.

이때 추가된 언어 규약에서 주목할 만한 특징은 메모리 관리가 C 언어보다 수월해졌다는 점이다. C++은 C와 달리 자동으로 메모리를 할당하거나 해제할 수 있는 new, delete 키워드를 지원했다. 또한 레퍼런스라는 개념을 통해 포인터를 더 편리하게 다룰 수 있게 되었으며, 가상 함수와 연산자 오버로딩을 지원하기 시작하면서 객체 지향 언어의 특징인 다형성을 완전하게 지원할 수 있게 되었다.

<br>

## C++ 표준 제정

초반에는 컴파일러마다 문법이 다르게 해석되는 문제가 있었다. 이에 따라 1998년에 C++의 첫 번째 국제 표준 언어 규격인 C++98이 제정되었다. C++98의 가장 큰 특징은 템플릿을 활용한 메타 프로그래밍과 이를 활용한 STL의 도입이다.

# C++ 프로그램 빌드 과정

</aside>

컴파일이란 C++ 소스 코드를 컴퓨터가 이해할 수 있는 코드로 변경하는 과정을 말한다. 그리고 각각의 소스 파일을 컴파일하여 만든 오브젝트 파일들을 하나의 실행 파일로 묶는 과정을 링크라고 한다. 컴파일과 링크를 거쳐 프로세서가 실행할 수 있는 파일이 만들어진다.

C++는 컴파일하기 전에 전처리 과정을 거친다. 전처리는 소스 파일이 컴파일되기 전에 소스코드를 변경하거나 확장하는 등의 작업을 의미한다.

요약하면 C++로 작성한 코드는 `#include`와 같은 지시문을 전처리하고 이렇게 준비된 소스 코드를 오브젝트로 컴파일한다. 그리고 소스에 포함한 각종 라이브러리와 오브젝트 코드를 연결하는 링크를 거쳐 최종 실행 파일을 만든다. 이 과정을 통틀어 빌드라고 한다.

# C++ 언어의 특징

</aside>

프로그래밍 언어는 저마다 고유한 특징이 있다. 같은 객체지향 언어라도 설계 철학이 다르고 사용처가 다르다.

<br>

## C++ 언어의 성공 요인

- 낮은 수준 액세스와 추상화
- 유용한 도구
- 시점
- 비독점
- 안정성
- 발전

<br>

## 객체지향 프로그래밍

C++는 객체지향 프로그래밍 언어이다. 프로그래밍에서 객체(object)는 ‘처리할 데이터와 처리 방법을 객체라는 논리 단위로 모두 포함하는 것’으로 간단히 설명할 수 있다.

<br>

## 저수준 접근과 메모리 직접 관리

C++이 가진 또 다른 특징은 저수준 접근이 가능하다는 것이다. 저수준 접근이란, 프로그램이 동작하는 시스템의 하드웨어에 직접 접근하여 제어하는 것을 말한다. 또한 메모리도 개발자가 프로그래밍으로 직접 관리할 수 있었다.

저수준 접근과 메모리 직접 관리는 불필요한 과정을 줄여 매우 효율적인 프로그램을 개발할 수 있게 한다.

<br>

## 데이터 추상화와 범용성

프로그램으로 다양한 문제를 해결한다는 것은 다양한 형태의 데이터와 구조를 다뤄야 한다는 의미이다. 크게 다음처럼 두 가지를 생각해볼 수 있다.

- 다양한 데이터와 구조를 처리할 수 있는 모든 방법을 제시한다.
- 한 가지 방법으로 여러가지 데이터와 구조를 처리한다.

<br>

C++은 두 번째 방식을 채택해 다양한 데이터와 구조를 한 가지 방식으로 쉽게 처리할 수 있게 했다. 이를 범용성이라고 하며 C++ 언어는 객체를 추상화하는 방법으로 범용성을 구현한다.

# C++ 개발 생태계

</aside>

개발 생태계는 프로그래밍 언어가 발전할 수 있는 밑바탕이다.

<br>

## 표준 명세

C++ 규약은 1998년 ISO 표준으로 채택된 이후 새로운 규약이 지속해서 추가되고 있다. C++ 언어는 C++11부터 3년 마다 새로운 표준안을 내놓으며 새로운 요구사항과 시대 흐름을 반영하고 있다.

C++11부터를 흔히 ‘모던 C++’이라고 한다.

<br>

## 커뮤니티

프로그래밍 언어가 발전하고 유지되려면 표준 활동 외에도 대중에게 알리고 사용자끼리 유대를 장려하는 커뮤니티 활동 역시 필요하다.

- CppCon
- Cpp 핵심 가이드라인
- C++ 코리아

<br>

## 개발 환경과 라이브러리

**통합 개발 환경**

- 비주얼 스튜디오
- 비주얼 스튜디오 코드
- 엑스코드
- 이클립스

<br>

**컴파일러와 디버거**

- GCC와 GDB
- Clang과 LLDB
- DPC++/C++

<br>

**라이브러리**

현대의 소프트웨어는 규모도 크고 복잡한 기술을 바탕으로 설계되어 전체를 혼자 만들 수는 없다. 라이브러리나 프레임워크처럼 이미 개발된 공통의 기능을 가져와 사용해야 한다.

많이 사용하는 C++ 라이브러리는 다음과 같다.

- C++ 표준 라이브러리
- Boost C++(boost.org)
- OpenCV(opencv.org)
- POCO((pocoproject.org)
- 텐서플로(tensorflow.org)
- 언리얼 게임 엔진(unrealengine.com)
